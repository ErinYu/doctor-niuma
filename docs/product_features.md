**《医生的数字牛马 Agent 产品开发需求文档（PRD）》**

**版本**：1.0（基于 GitHub 热门开源项目 NanoClaw 二次开发）  
**文档日期**：2026 年 3 月  
**产品代号**：DoctorNiuMa（医生的数字牛马）  
**定位**：对标《OpenAI for Financial Services：全套金融工具链》，专为中国大陆医生打造的企业级/个人级 AI 个人助理套件。由轻量级 Claude Agent SDK（NanoClaw 核心）驱动，运行于安全容器中。核心理念：**让医生彻底摆脱“牛马”式琐碎劳动**，专注临床决策、科研创新与职业成长。  
**核心组件映射**：  
- **医疗数据集成**：通过“Medical Context Protocol（MCP 医疗版）”或安全文件/API 接入自有数据（EMR、LIS、PACS、个人病例库）。  
- **可复用 Skills**：针对高频医疗任务的预构建技能模板（Skills 文件夹）。  
- **结构化输出**：利用 Apps + Research 功能，从病历、影像报告、电话会议录音、PPT、Excel 中提取信息，生成可导出 PDF/Word/医院系统模板的结构化报告。  

### 1. 背景推演：中国大陆医生日常工作、流程、诉求与痛点分析

**日常工作流程（反复推演典型场景）**：  
- **三级医院内科主治医师**：早 7:30 查房（20-40 床）→ 写病程记录 → 9:00 门诊（30-60 患者/半天，5-10 分钟/人）→ 开医嘱/检查 → 下午手术/会诊/科研 → 晚上补病历、追文献。  
- **外科住院医师**：术前讨论+手术记录+术后随访+值班。  
- **影像/检验科**：高强度阅片/报告书写。  
- **基层全科**：全科诊疗+公共卫生+慢病管理。  
全天工作量普遍 50-70 小时/周，病历书写占比 30-40% 时间。  

**职业发展诉求与卡点**：  
- **晋升路径**（住院医师→主治→副主任医师→主任医师）：需临床工作量（病例/手术数）、论文（核心期刊/SCI）、课题、教学、基层服务（部分省份要求）。卡点：论文撰写、病例积累、继续教育学分追踪、职称材料整理。  
- **异质性**：  
  - **初级 vs 高年资**：初级重文档+学习；高年资重决策+带教+科研。  
  - **科室差异**：内科需指南更新+药物交互；外科需手术规划+记录；肿瘤科需方案比对+文献；影像科需报告辅助。  
  - **医院层级**：三甲重科研；基层重实用工具+患者沟通。  
  - **地域**：一线城市医生更需多点执业管理；基层医生更需远程会诊支持。  

**核心痛点总结**（数据来源：医师白皮书、医院调研、职称政策）：  
- 文档负担重（病历书写、随访记录）。  
- 知识更新压力（每日新指南、药物、文献）。  
- 职业发展碎片化（晋升材料、CME 学分、课题追踪）。  
- 数据孤岛+隐私合规（PIPL、《数据安全法》、医院 HIS 不允许公网传输 PHI）。  
- 消息触达碎片化（微信群、患者随访）。  
-  burnout 高发（医闹恐惧、超负荷）。  

**设计约束**：  
- **消息触达**：首选微信/企业微信（医生 99% 使用场景），支持主动推送、语音交互、群聊机器人。  
- **数据隔离**：NanoClaw 原生容器隔离 + 本地部署 + 加密；每位医生独立容器/向量库，PHI 永不离开本地/医院私有云。  
- **个性化**：医生 Profile（科室、职称、偏好风格）+ 反馈循环 + 长期记忆（NanoClaw CLAUDE.md + SQLite + 向量 RAG）。  
- **长期记忆**：跨会话保留个人病例库、偏好模板、历史决策。  
- **高可用成熟产品**：离线优先、本地 LLM 兜底、自动备份、监控仪表盘、医疗合规模块化验证。  

### 2. 最有价值的 10 个医生版个人助理需求点（逐步推演结果）

基于上述推演，按**日常效率提升（60%）+ 职业发展加速（30%）+ 异质性适配（10%）** 优先级排序，最终锁定以下 10 个高价值、可复用 Skill（每个 Skill 对应一个 NanoClaw skill 文件夹，可独立启用/定制）：

1. **AI 病历自动生成与质控**（最高价值，覆盖 80% 医生日常）  
   语音/拍照/粘贴 → 结构化入院记录、病程、手术记录、出院小结；自动拉取 LIS/PACS 数据；合规检查（病历书写基本规范）；导出 HIS 或 PDF。异质性：外科加手术记录模板，影像科加报告模板。

2. **个性化文献/指南实时更新助手**  
   每日/按病例推送 CNKI、Wanfang、PubMed、中华医学会指南摘要；RAG 检索；“这个病例最新指南怎么说？”一键问答。长期记忆：记住医生研究方向。

3. **临床决策支持（差分诊断 + 用药审查）**  
   输入症状/化验/影像描述 → 给出 3-5 个鉴别诊断 + 证据链（引用指南）；药物交互/禁忌/剂量；免责声明 + “建议结合临床”。支持离线本地模型。

4. **科研论文助手**  
   文献检索 → 结构化摘要 → 论文初稿生成（中英文）→ 参考文献管理 → 统计分析（简单代码解释器）；自动生成职称所需病例总结。

5. **患者随访与宣教管理器**  
   生成个性化宣教材料（微信图文/语音）；自动随访提醒；随访记录结构化入库；隐私加密传输。

6. **职称晋升 & 职业发展追踪器**  
   输入职称要求 → 自动追踪已完成病例数/手术数/论文/学分；生成晋升材料包（病例汇总、CV 章节、基层服务证明模板）；提醒截止日期。

7. **行政报告与统计自动化**  
   从 Excel/病历中提取 KPI（门诊量、出院人数、DRG 等）；生成科室月报、绩效报表、教学 PPT；支持医院绩效考核格式。

8. **多模态报告智能解读**  
   上传影像/PDF/Excel → 总结关键异常 + 趋势图 + 建议；支持 PACS 截图或导出文件。特别适合影像科、肿瘤科。

9. **个人知识库 & 长期记忆引擎**  
   上传个人笔记、病例库、偏好模板 → 向量 RAG 永久记忆；跨设备同步（加密）；“回忆我上次类似病例怎么处理的？”。

10. **教学与多点执业支持**  
    病例教学 PPT 自动生成；小讲课大纲、考题；多点执业病历跨机构安全摘要（脱敏）；值班交班总结。

这些需求点已充分考虑**异质性**（通过 Profile 动态加载子模板）、**消息触达**（微信主动推送每日摘要、提醒）、**数据隔离**（容器 + 本地向量库）、**个性化**（反馈学习 + 风格记忆）、**长期记忆**（NanoClaw 原生 + RAG）。

### 3. 完整产品开发需求文档（可直接落地实施）

#### 3.1 产品架构（基于 NanoClaw 二次开发）

- **核心框架**：Fork https://github.com/qwibitai/nanoclaw（轻量 ~500 行核心代码，容器隔离、安全、记忆、定时任务、Skills 扩展性极强）。  
- **新增通道**：Wechaty / 企业微信官方 API Skill（首选消息入口，支持语音、群聊、文件接收）。  
- **LLM 层**：混合模式  
  - 本地优先：Ollama + 通义千问 / DeepSeek / 医用微调模型（数据主权）。  
  - 备选：Anthropic Claude（海外医生）或安全合规云端（医院私有部署）。  
- **数据层**：  
  - 本地 SQLite + Chroma/FAISS 向量库（个人病例 KB）。  
  - MCP 医疗版协议：医生拖拽文件/安全 API Connector（HIS/LIS/PACS 读-only）。  
- **安全隔离**：每个医生独立 Docker/Apple Container；PHI 零公网传输；审计日志。  
- **Agent Swarm**：复杂任务（如“帮我写一篇肿瘤免疫治疗综述”）拆分为 Search Agent + Draft Agent + Review Agent。  
- **部署方式**：  
  - 个人版：医生笔记本 Docker 一键部署。  
  - 医院版：Kubernetes 私有云 + 统一管理后台（可选）。  

#### 3.2 非功能需求（成熟产品标准）

- **高可用**：99.9% 在线；离线模式（本地模型 + 缓存）；自动备份（加密）。  
- **性能**：门诊场景 < 3 秒响应；语音转文字 < 2 秒。  
- **合规**：通过《个人信息保护法》《数据安全法》《病历书写基本规范》；提供第三方等保/医疗器械软件认证支持材料。  
- **可扩展性**：Skills 热加载；医生可自行编写简单技能（Markdown 定义）。  
- **国际化**：中英双语，优先中文界面。  
- **监控**：内置 Prometheus + Grafana（匿名使用统计）；错误自动上报（脱敏）。  

#### 3.3 开发路线图（3 个阶段，6-12 个月交付成熟产品）

**Phase 1：MVP（3 个月）**  
- NanoClaw 基础 + WeChat 通道 + Skills 1、2、9（病历、文献、个人 KB）。  
- 本地 LLM 集成 + 基本 RAG。  
- 交付：可内测版，10 名医生验证时间节省 >30%。

**Phase 2：核心功能全覆盖（4-6 个月）**  
- 上线全部 10 个 Skill + 结构化输出 + 职称追踪。  
- 医院 API Connector 示例（东软/万达 HIS）。  
- 安全审计 + 离线优化。

**Phase 3：成熟产品 & 企业化（3 个月）**  
- 医院多用户管理后台。  
- 医疗验证报告 + 合规文档。  
- App 端（微信小程序包装）。  
- 持续更新机制（Skills 市场）。

#### 3.4 测试与上线计划

- **测试**：功能测试 + 医疗专家盲测（准确率、合规性）+ 压力测试 + 渗透测试。  
- **上线**：Docker 一键脚本 + 详细中文部署文档（含 WeChat 配置）。  
- **运维**：自动更新脚本、备份策略、社区 Slack/微信群支持。  
- **KPI 成功标准**：  
  - 医生反馈：每周节省 10+ 小时文档/文献时间。  
  - 采用率：首年 1000+ 个人用户或 10 家医院试点。  
  - 保留率 >85%，NPS >70。  

#### 3.5 风险与缓解

- 隐私风险 → 容器隔离 + 本地优先 + 法律免责声明。  
- 医疗责任 → 所有输出加“仅供参考，医生最终决策” + 日志可追溯。  
- 集成难度 → 提供文件拖拽 + 标准 API 示例，医院可自行适配。  
- 模型幻觉 → RAG + 引用来源 + 医生编辑后确认流程。
