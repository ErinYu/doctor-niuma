# AI 病历自动生成与质控 (Medical Record Skill)

该技能帮助医生快速生成规范的医疗文书并进行质量控制，减轻书写负担。

## 核心功能

1. **结构化提取**：从医生输入的零散语音记录、文本、化验单、影像报告片段中，自动提取出主诉、现病史、既往史、体格检查、辅助检查等关键信息。
2. **病历生成**：根据提取的信息，生成标准化的：
   - 入院记录 (Admission Note)
   - 首次病程记录 (First Progress Note)
   - 日常病程记录 (Daily Progress Note)
   - 手术记录 (Operative Note)
   - 出院小结 (Discharge Summary)
3. **病历质控 (Quality Control)**：
   - 检查是否有逻辑矛盾（如“未见异常”与具体病理描述冲突）。
   - 检查是否遗漏关键要素（如：主诉是否包含症状和持续时间）。
   - 确保遵循《病历书写基本规范》。

## 使用工作流

当你接收到医生的零散输入（或提示“帮我写个入院记录/病历”等）时，请执行以下步骤：

### 第一步：信息解析与结构化
分析医生提供的所有输入数据，将它们归类到病历的各个章节中。如果发现关键信息（如主诉的持续时间）缺失，可以作为待确认项标记。

### 第二步：输出结构化病历模板
输出符合规范的病历。使用 Markdown 格式清晰展示。例如对于入院记录：

```markdown
**主诉**：[症状] + [持续时间]
**现病史**：起病情况、症状特点、伴随症状、诊疗经过、一般情况等。
**既往史**：平素健康状况、疾病史、传染病史、手术外伤史等。
**个人史**：出生地、生活习惯等。
**家族史**：家族遗传病史等。
**体格检查 (查体)**：T, P, R, BP, 专科查体阳性体征等。
**辅助检查**：重要的检验/影像学结果。
**初步诊断**：
1. [主要诊断]
2. [次要诊断]
```

### 第三步：自动质控报告 (QC)
在病历下方，附上一段简短的质控分析：
- **完整性检查**：[通过/发现遗漏：未提及过敏史]
- **逻辑一致性**：[通过/警告：查体血压偏高但无高血压病史，请确认]
- **规范性提示**：[符合基本规范要求]

## 动态模板适配
根据医生 Profile 或指明的科室（如外科），自动加载对应的专科病历模板（例如：外科增加专科检查及术前讨论重点，肿瘤科增加 TNM 分期及病理结果等）。

## 隐私与安全
- 输出的病历不应包含真实的患者可识别信息（如真实姓名、身份证号），除非医生明确要求在本地加密生成。
- 提醒提示：生成结果仅供参考，医生需最终核对签字。
