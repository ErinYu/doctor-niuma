# 临床决策支持 (Medical Decision Support Skill)

本技能利用 RAG (检索增强生成) 和医学知识图谱，为医生提供诊断思路和用药安全性审核。

## 核心功能

1.  **鉴别诊断 (Differential Diagnosis, DDx)**：
    - 输入患者的主要症状、体格检查阳性体征、关键化验指标。
    - 输出 3-5 个可能的鉴别诊断，并附带每个诊断的“支持点”与“不支持点”。
    - 提供“下一步建议检查”以辅助排除。
2.  **用药审查 (Medication Safety)**：
    - 输入拟使用的药物清单、患者过敏史、既往基础疾病。
    - 检查药物间的相互作用 (DDI)。
    - 检查禁忌症（如肾功能不全患者的药物减量）。
    - 检查重复用药。

## 使用工作流

### 第一步：诊断推理分析
分析医生提供的临床表现，优先考虑高致残率、高致死率的诊断。

```markdown
### 临床诊断思路建议
**初步印象**：[推测的诊断方向]
**鉴别诊断表**：
| 拟诊疾病 | 支持依据 | 不支持依据/需排除点 | 风险等级 |
| :--- | :--- | :--- | :--- |
| [疾病A] | [临床表现1] | [指标X正常] | 高/中/低 |
| [疾病B] | [影像表现2] | [症状不典型] | 高/中/低 |

**建议进一步检查**：[如：急诊床旁超声、D-二聚体等]
```

### 第二步：用药安全性审查
当医生提及处方或治疗方案时，自动进行安全审查。

```markdown
### 用药安全性提醒
- **⚠️ 相互作用**：[药物A] 与 [药物B] 合用可能增加出血风险，请注意。
- **🚫 禁忌症提醒**：患者存在[慢性肾脏病]，[药物C]需减量或停用。
- **✅ 重复用药检查**：未发现重复用药。
```

## 责任与边界
- **强制提示**：所有输出必须标注“本建议仅供临床思维参考，不能代替医生的最终决策”。
- **引用依据**：尽可能引用最新指南或药典作为依据。
- **离线模型支持**：在涉及隐私数据时，优先调用本地 Ollama 部署的医疗专用模型。
