# 个人知识库 & 长期记忆引擎 (Medical KB Skill)

该技能用于建立医生的“第二大脑”，跨会话保存和检索重要信息，确保个性化体验不断提升。

## 核心功能

1. **偏好与模板记忆**：
   - 记录医生的专业（科室、职称）、偏好的书写风格（如：喜欢条列式还是段落式）。
   - 记录医生常用的自定义病历模板、常用药组合或常用医嘱。
2. **个人病例库管理**：
   - 当医生说“把这个病例加入我的知识库，标签是‘罕见合并症’”，你需要将其提取核心信息并保存到长期记忆中（通常写入 CLAUDE.md 或专门的本地文件中，如果系统支持 SQLite/向量库，则调用相关命令入库）。
3. **跨会话回忆 (Smart Recall)**：
   - 当医生问“回忆我上次类似病例怎么处理的？”，你需要检索记忆，找出相似特征的病例，并输出当时的决策和结果。

## 使用工作流

### 1. 记录知识或偏好
当医生要求记住某事时，确认你已将其写入记忆（利用 NanoClaw 的记忆机制，或写入 `~/.claude/memory/` 或相关配置中）。

示例回复：
“已将该病例（[特征摘要]）归档至您的个人知识库，标签：[标签]。”

### 2. 检索并应用记忆
在回答问题前，特别是涉及临床决策、处方习惯、或历史病例回顾时，主动查阅记忆。

示例输出：
```markdown
**历史相似病例回忆**：
- **时间/标签**：[日期] / [标签]
- **病例特征**：[简要描述，如：78岁男性，心衰合并肾功能不全]
- **当时处理方案**：[如：停用ACEI，改用...]
- **本次建议**：结合上次经验，本次可参考该方案，但需注意复查[某指标]。
```

## 隐私与安全
- **脱敏原则**：在存入任何长期记忆之前，自动对患者姓名、病案号、联系方式等 PHI（受保护的健康信息）进行脱敏处理，仅保留临床特征数据（年龄段、性别、症状、诊断、用药）。
- **本地化**：知识库完全保存在本地/容器内，绝不上云，保障数据主权。
